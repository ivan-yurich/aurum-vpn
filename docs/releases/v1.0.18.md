# Aurum VPN Android 1.0.18

Date: 2026-05-25

## What changed

- Moved sniffing to Android TUN and mixed inbounds, closer to the NekoBox config builder.
- Added inbound `domain_strategy`, DNS `independent_cache`, and FakeDNS `disable_cache`.
- Changed NaiveProxy startup order to HTTPS CONNECT first, native Naive second.
- Added a short Naive tunnel throughput probe: if the first mode connects but is very slow, the app retries with the next mode.
- Kept VLESS `packet_encoding: xudp`, Hysteria2 import, and Hysteria import from 1.0.17.

## Verification

- `flutter test`
- `flutter analyze`
- `flutter build apk --release --split-per-abi`
- `apksigner verify --verbose` passed for all split APKs with APK Signature Scheme v2.

## Server check

- Caddy is active and includes `http.handlers.forward_proxy`.
- Server-side BBR/fq is enabled.
- Server outbound download test reached about 45 MB/s.
- The server has 1 CPU, so Caddy/Naive TLS load can still become a bottleneck under concurrent use.
